Реализованные алгоритмы:
1. Пошаговый алгоритм (Step-by-step)
Принцип работы: Аппроксимация линии путем равномерного распределения шагов по большей из координатных осей.
Характеристики:
Использует деление и округление
Поддерживает дробные координаты
Сложность: O(max(|Δx|, |Δy|)

2. Алгоритм ЦДА (Цифровой дифференциальный анализатор)
Принцип работы: Улучшенная версия пошагового алгоритма с предварительным вычислением приращений.
Характеристики:
Меньше вычислений на точку по сравнению с пошаговым
Сложность: O(max(|Δx|, |Δy|))

3. Алгоритм Брезенхема для линий
Принцип работы: Использование целочисленной арифметики для минимизации ошибки.
Характеристики:
Только целочисленные операции (+, -, сравнения)
Не использует деление и умножение
Наиболее эффективный алгоритм для линий
4. Алгоритм Брезенхема для окружностей
Принцип работы: Построение окружности с использованием симметрии восьми октантов.
Характеристики:
Генерация 8 точек за итерацию
Целочисленные вычисления
Симметричное построение

Интерфейс приложения:
1. Панель управления (левая часть):
-Выбор алгоритма: выпадающий список с 4 вариантами
-Координаты:
X1, Y1 - начальная точка (центр для окружности)
X2 - конечная X или радиус окружности
Y2 - конечная Y (не используется для окружности)
-Кнопки управления:
"Построить" - запуск анимации растеризации
"Сбросить вид" - возврат к исходному масштабу
2. Холст (правая часть):
-Координатная система:
Центр в середине холста
Ось X направлена вправо
Ось Y направлена вверх
-Сетка: ячейки 20×20 пикселей
-Подписи осей: целочисленные значения координат
-Информация о времени: в правом нижнем углу
3. Навигация:
Перемещение: зажать ЛКМ и двигать мышь

Инструкция по использованию:
Шаг 1: Настройка параметров
Выберите алгоритм из выпадающего списка
Введите координаты:
Для линии: начальная (X1,Y1) и конечная (X2,Y2) точки
Для окружности: центр (X1,Y1) и радиус (X2)

Шаг 2: Запуск построения
Нажмите кнопку "Построить"
Наблюдайте анимацию процесса растеризации
После завершения смотрите время выполнения в правом нижнем углу
Шаг 3: Навигация
Перемещение: Зажмите левую кнопку мыши на холсте и перемещайте
Сброс: Нажмите "Сбросить вид" для возврата к исходному положению

Пример вычислений:
1. Пошаговый алгоритм (1,1)-(2,8)
dx = 1, dy = 7
|dx| < |dy| → основная ось Y

Инкремент по X при шаге по Y:
x_inc_per_y = dx/dy = 1/7 ≈ 0.142857

y = 1 → x = 1 → (1,1)
y = 2 → x = 1 + 0.142857 = 1.142857 → округление → (1,2)
y = 3 → x = 1.142857 + 0.142857 = 1.285714 → (1,3)
y = 4 → x = 1.285714 + 0.142857 = 1.428571 → (1,4)
y = 5 → x = 1.428571 + 0.142857 = 1.571428 → округление → (2,5)
y = 6 → x = 1.571428 + 0.142857 = 1.714285 → (2,6)
y = 7 → x = 1.714285 + 0.142857 = 1.857142 → (2,7)
y = 8 → x = 1.857142 + 0.142857 = 2.000000 → (2,8)

2. Алгоритм ЦДА (1,1)-(2,8)
dx = x2 - x1 = 2 - 1 = 1
dy = y2 - y1 = 8 - 1 = 7
steps = max(|dx|, |dy|) = max(|1|, |7|) = 7

x_increment = dx/steps = 1/7 ≈ 0.142857
y_increment = dy/steps = 7/7 = 1

x = 1, y = 1

Шаг 0: x=1, y=1 → (1,1)
Шаг 1: x=1.142857, y=2 → (1,2)
Шаг 2: x=1.285714, y=3 → (1,3)
Шаг 3: x=1.428571, y=4 → (1,4)
Шаг 4: x=1.571428, y=5 → (2,5)
Шаг 5: x=1.714285, y=6 → (2,6)
Шаг 6: x=1.857142, y=7 → (2,7)
Шаг 7: x=2.000000, y=8 → (2,8)

3. Алгоритм Брезенхема для линии (1,1)-(2,8)
 x = 1, y = 1
x2 = 2, y2 = 8

dx = |x2 - x| = |2 - 1| = 1
dy = |y2 - y| = |8 - 1| = 7
sx = (x < x2) ? 1 : -1 = 1
sy = (y < y2) ? 1 : -1 = 1

err = dx - dy = 1 - 7 = -6

Шаг 0: точка (1,1), err=-6
e2 = 2*err = -12
e2 > -dy? (-12 > -7) = ложь
e2 < dx? (-12 < 1) = истина → err=err+dx=-5, y=y+sy=2

Шаг 1: точка (1,2), err=-5
e2 = -10
e2 > -dy? (-10 > -7) = ложь
e2 < dx? (-10 < 1) = истина → err=err+dx=-4, y=3

Шаг 2: точка (1,3), err=-4
e2 = -8
e2 > -dy? (-8 > -7) = ложь
e2 < dx? (-8 < 1) = истина → err=err+dx=-3, y=4

Шаг 3: точка (1,4), err=-3
e2 = -6
e2 > -dy? (-6 > -7) = истина → err=err-dy=-10, x=x+sx=2
e2 < dx? (-6 < 1) = истина → err=err+dx=-9, y=5

Шаг 4: точка (2,5), err=-9
e2 = -18
e2 > -dy? (-18 > -7) = ложь
e2 < dx? (-18 < 1) = истина → err=err+dx=-8, y=6

Шаг 5: точка (2,6), err=-8
e2 = -16
e2 > -dy? (-16 > -7) = ложь
e2 < dx? (-16 < 1) = истина → err=err+dx=-7, y=7

Шаг 6: точка (2,7), err=-7
e2 = -14
e2 > -dy? (-14 > -7) = ложь
e2 < dx? (-14 < 1) = истина → err=err+dx=-6, y=8

Шаг 7: точка (2,8) - конечная точка

4. Алгоритм Брезенхема для окружности
x = 0, y = 5
d = 3 - 2*r = 3 - 10 = -7

Октант 1: (xc+x, yc+y) = (1,6)
Октант 2: (xc-x, yc+y) = (1,6)
Октант 3: (xc+x, yc-y) = (1,-4)
Октант 4: (xc-x, yc-y) = (1,-4)
Октант 5: (xc+y, yc+x) = (6,1)
Октант 6: (xc-y, yc+x) = (-4,1)
Октант 7: (xc+y, yc-x) = (6,1)
Октант 8: (xc-y, yc-x) = (-4,1)

Итерация 1:
y >= x? 5 >= 0 = истина
x++ = 1
d > 0? (-7 > 0) = ложь → d = d + 4*x + 6 = -7 + 4 + 6 = 3

Итерация 2:
y >= x? 5 >= 1 = истина
x++ = 2
d > 0? (3 > 0) = истина → y-- = 4, d = d + 4*(x-y) + 10 = 3 + 4*(2-4) + 10 = 3 - 8 + 10 = 5

Итерация 3:
y >= x? 4 >= 2 = истина
x++ = 3
d > 0? (5 > 0) = истина → y-- = 3, d = d + 4*(x-y) + 10 = 5 + 4*(3-3) + 10 = 15

Итерация 4:
y >= x? 3 >= 3 = истина
x++ = 4
d > 0? (15 > 0) = истина → y-- = 2, d = d + 4*(x-y) + 10 = 15 + 4*(4-2) + 10 = 15 + 8 + 10 = 33

Итерация 5:
y >= x? 2 >= 4 = ложь → завершение

Временные характеристики алгоритмов:
1. Для коротких отрезков (≤10 точек):
Все алгоритмы примерно равны по скорости
Разница незначительна (<1 мс)

2. Для длинных отрезков (>100 точек):
Брезенхема - самый быстрый
ЦДА - средняя скорость
Пошаговый - самый медленный
3. По типам линий:
Горизонтальные/вертикальные линии:
-Брезенхема и ЦДА одинаково быстры
-Пошаговый медленнее на 20-30%
Линии под углом ~45°:
-Брезенхема быстрее всех
-ЦДА на 30% медленнее
-Пошаговый на 50% медленнее
Очень крутые линии (|dy| >> |dx|):
-Брезенхема и ЦДА примерно равны
-Пошаговый самый медленный
